블로그 에러 사항들

Updated on 2025-04-03

블로그 만들 때 에러들

  • *.txt?_rsc=*** 404 (Not Found)

화면이 안나오거나 하는건 아니었는데, 블로그에 접속하면 이 콘솔 에러가 계속 발생했다.

결론적으로는 next.config.ts 파일에서 trailingSlash: true 옵션을 추가해주니 에러가 사라졌다.

trailingSlash: false(기본값)인 경우, Next.js는 URL 끝에 슬래시를 추가하지 않는다.
trailingSlash: true를 설정하면, Next.js가 모든 URL 끝에 슬래시를 추가한다.

예를 들어:

  • /about → /about/
  • /posts/1 → /posts/1/

현재 정확한 이유는 모르겠지만

  • https://kajing1077.github.io/wangzzang/
  • https://kajing1077.github.io/wangzzang

이와같이, 뒤의 slash로 인해서 경로가 잘못되어서 404 error가 발생하는 것으로 보인다.

구글링 결과 짐작하는 바로는..
GitHub Pages는 정적 파일을 서빙할 때, 디렉토리 구조를 기반으로 URL을 해석한다.
github pages는 Jekyll 3를 사용하는데 이에 관련된 문제라고 생각된다.

관련 포스팅

관련 자료

In Jekyll 2, any URL constructed from the permalink: field had a trailing slash (/) added to it automatically. Jekyll 3 no longer adds a trailing slash automatically to permalink: URLs. This can potentially result in old links to pages returning a 404 error.
